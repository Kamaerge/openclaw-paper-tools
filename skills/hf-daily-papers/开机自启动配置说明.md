## 📋 开机自启动功能

系统重启后，会自动执行以下检查：

1. **OpenClaw CLI 可用性检查**
   - 验证 OpenClaw 是否正常工作
   - 显示版本信息

2. **Python 环境检查**
   - 验证 Python 3.13.5 是否可用
   - 路径: `C:\MiniConda\python.exe`

3. **计划任务状态检查**
   - 验证每日推送任务是否正常
   - 显示上次/下次运行时间
   - 检查运行结果

4. **工作目录完整性检查**
   - 验证必需文件是否存在:
     - `generator.py`
     - `multi_source_fetcher.py`
     - `run_and_send.ps1`
     - `run_and_send_task.ps1`

5. **网络连接检查**
   - 测试 arXiv.org 连接
   - 确保能正常获取论文

## ⚙️ 高级配置（可选）

### 方案 A: 系统级开机自启动（需要管理员权限）

如果需要系统级别的开机启动任务（更可靠，但需要管理员权限）：

```powershell
# 以管理员身份运行 PowerShell，然后执行：
cd D:\Project\openclaw-paper-tools\skills\hf-daily-papers
.\register_startup_admin.ps1 -DelayMinutes 3
```

**优点**:
- 更高优先级
- 支持多用户环境
- 不依赖用户登录

**缺点**:
- 需要管理员权限
- 配置复杂

### 方案 B: 开机自动推送（当前已禁用）

如果希望开机时自动运行一次论文推送（而不是等到每天8点）：

1. 编辑 `startup_check.ps1`
2. 找到这一行：`if ($env:AUTO_RUN_ON_STARTUP -eq '1')`
3. 重新注册时添加参数：
   ```powershell
   .\register_startup_user.ps1 -AutoRunOnStartup
   ```

## 🔧 故障排查

### 问题 1: 开机后没有运行
**检查步骤**:
1. 确认快捷方式存在：
   ```powershell
   Test-Path "$env:APPDATA\Microsoft\Windows\Start Menu\Programs\Startup\OpenClaw-Papers-Startup.lnk"
   ```
2. 检查启动日志：
   ```powershell
   Get-Content "D:\Project\openclaw-paper-tools\skills\hf-daily-papers\startup_$(Get-Date -Format 'yyyy-MM-dd').log"
   ```

### 问题 2: 权限错误
**解决方案**:
- 使用用户级别配置（已配置）：`.\register_startup_user.ps1`
- 如需系统级别，以管理员身份运行：`.\register_startup_admin.ps1`

### 问题 3: 每天 8 点没有推送
**检查步骤**:
1. 验证计划任务状态：
   ```powershell
   Get-ScheduledTask -TaskName "OpenClaw-HF-Daily-Papers-Feishu"
   Get-ScheduledTaskInfo -TaskName "OpenClaw-HF-Daily-Papers-Feishu"
   ```
2. 检查上次运行结果
3. 手动测试运行：
   ```powershell
   schtasks /Run /TN "OpenClaw-HF-Daily-Papers-Feishu"
   ```
