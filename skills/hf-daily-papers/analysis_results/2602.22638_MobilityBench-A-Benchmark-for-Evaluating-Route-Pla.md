## MobilityBench 论文分析

### 1. 核心贡献

论文提出了 **MobilityBench**，这是首个大规模评估 LLM 驱动的路径规划智能体的基准测试。其核心创新包括：
- 基于高德地图真实用户语音查询构建，覆盖全球 350+ 城市的真实出行场景
- 设计了**确定性 API 重放沙箱**，解决了实时地图服务非确定性的难题，确保评估可复现
- 提出**多维评估协议**，同时考察结果有效性、指令理解、规划能力、工具使用和效率

### 2. 技术方法

- **数据构建**：从高德地图收集大规模匿名用户查询，经过多阶段过滤和意图分类，构建包含 11 种任务场景的 taxonomy
- **重放沙箱**：预录制 API 响应并缓存，评估时拦截智能体调用并从缓存解析，确保确定性
- **结构化真值**：按领域专家定义的 SOP（标准操作程序）构建，包含最小工具调用序列和中间证据
- **多维评估**：验证 API 调用正确性、约束满足度、指令理解、规划质量、工具使用效率

### 3. 实验亮点

- 在 4 类任务家族上评估了多种 LLM 路径规划智能体
- 发现当前模型在 **Basic Information Retrieval** 和 **Basic Route Planning** 任务上表现良好
- **Preference-Constrained Route Planning**（偏好约束路径规划）表现较差，如避开高速、减少换乘等需求
- 发布完整 benchmark 数据、评估工具包和文档于 GitHub

### 4. 相关工作

- **ToolBench**（通用 API 调用）、**τ-bench**（交互式评估）等通用工具智能体基准
- **TravelPlanner**、**TravelBench** 等旅行规划基准（但仅关注高层次行程生成）
- 传统路径规划算法（Dijkstra、A*）、LLM+传统规划器混合框架

### 5. 局限性

- 仅基于高德地图数据，可能存在地域偏差
- 严格要求"无澄清"假设，现实中用户常需多轮交互
- 偏好约束任务表现不佳，说明个性化出行需求仍是挑战
- 评估依赖预录制数据，可能无法完全反映真实动态交通场景

---

**总结**：MobilityBench 为 LLM 路径规划智能体提供了首个真实世界评估基准，其沙箱设计值得借鉴，但偏好约束任务暴露了当前模型的明显短板，是后续研究的重要方向。
