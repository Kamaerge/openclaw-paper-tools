## 📝 论文概述

**UXSim** 是一个新颖的混合用户搜索模拟框架，旨在解决传统规则模拟器过于简单、而独立LLM智能体又缺乏可验证 grounding 的问题。该框架发表于 CIKM 2025。

---

## 1. 核心贡献

- **混合架构创新**：首次将传统规则模拟器与LLM智能体深度融合，通过 **Oris 编排策略** 智能调度两者
- **声明式界面抽象层**：提出 Recipe + Blueprint 机制，使模拟器能直接操作真实网页 UI（Google Search、ChatGPT、Perplexity）
- **可解释的认知追踪**：Oris-A 策略让 LLM 输出结构化的"思维过程"，便于验证用户决策的合理性
- **开源易用**：提供 Web 界面，降低研究门槛

---

## 2. 技术方法

| 组件 | 功能 |
|------|------|
| **传统模拟器** | 查询生成、相关性判断、停止决策（基于规则/概率） |
| **认知智能体 (Cognitive Agent)** | LLM 驱动，具备 Perceive/Plan/Act/Reflect/Wonder 五种认知能力 |
| **Oris 编排策略** | Oris-S（规则）、Oris-ML（监督学习）、Oris-A（LLM推理） |
| **界面抽象层** | Recipe 映射 UI 元素，Blueprint 执行具体操作 |

---

## 3. 实验亮点

- 在 **USimAgent2.0** 数据集上训练 Oris-ML 策略（使用 LightGBM 分类器）
- 展示了在三种不同搜索界面（传统搜索、对话式、混合式）上的模拟能力
- 提供行为分析模块评估模拟真实性

---

## 4. 相关工作

- **传统模拟器**：SimIIR、RecSim、UserSimCRS
- **LLM 模拟器**：USimAgent、BASES
- **认知架构**：ACT-R、UBM 模型

---

## 5. 潜在局限

1. **LLM 一致性问题**：即使有约束引导，LLM 仍可能产生不稳定/非人类行为
2. **策略调优复杂**：混合框架的效果高度依赖 Oris 策略的设计质量
3. **Recipe 维护成本**：针对不同网站需手动编写/维护 UI 映射规则
4.**：用户行为 **评估挑战真实性评估本身仍是开放问题

---

**一句话总结**：UXSim 通过"传统模拟器提供结构化约束 + LLM 提供灵活推理"的互补设计，为搜索系统的用户模拟提供了一个灵活、可解释、更贴近真实人类行为的研究框架。
